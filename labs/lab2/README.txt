```// Example data structure based on your description
const courseIdx = 0; // The current course index
const id = "student123"; // The student ID

// Mock arrays for the grades
const allAssignments = { "student123": [[85, 90, 78]] };
const allQuizzes = { "student123": [[88, 92]] };
const allExams = { "student123": [[82, 85]] };

// Helper function to calculate average
const getAvg = (arr) => arr.length ? (arr.reduce((a, b) => a + b) / arr.length).toFixed(2) : 0;

// Data configuration for the rows
const categories = [
    { type: 'Assignments', weight: 0.30, grades: allAssignments[id][courseIdx] },
    { type: 'Quizzes', weight: 0.20, grades: allQuizzes[id][courseIdx] },
    { type: 'Exams', weight: 0.50, grades: allExams[id][courseIdx] }
];

let finalScore = 0;

// Start building the htmlOutput
let htmlOutput = `
<table border="1">
    <thead>
        <tr>
            <th colspan="4">Course ${courseIdx + 1}</th>
        </tr>
        <tr>
            <th>Type</th>
            <th>Grades</th>
            <th>Avg</th>
            <th>Weighted</th>
        </tr>
    </thead>
    <tbody>
`;

// Use template literals to generate rows
categories.forEach(cat => {
    const avg = getAvg(cat.grades);
    const weighted = (avg * cat.weight).toFixed(2);
    finalScore += parseFloat(weighted);

    htmlOutput += `
        <tr>
            <td>${cat.type} (${cat.weight * 100}%)</td>
            <td>${cat.grades.join(', ')}</td>
            <td>${avg}</td>
            <td>${weighted}</td>
        </tr>`;
});

// Add the total row at the bottom
htmlOutput += `
    </tbody>
    <tfoot>
        <tr>
            <th colspan="3">Final Score</th>
            <th>${finalScore.toFixed(2)}</th>
        </tr>
    </tfoot>
</table>
`;

(a) What is the scope of a const variable?


Answer: const is block scoped. This means that the variable can only be accessed inside the closest block of code denoted by curly braces {} which can be an if statement, a loop, or a function.


** (b) When were the let and const keywords added to JavaScript?**

Answer: These keywords were released in 2015. This feature is technically called ECMAScript 6 or ES6.

** (c) If you were to use var instead of let in your script(s), would your script still work? Why? Why not?** 
Answer: In general, the script would still work, but its behavior with respect to scope would be altered. Why: var is function scoped, whereas let is block scoped. Using var creates "hoisting," which causes bugs as variables are accessed before they are declared or variables escape loops. In a new ES6 script, it is recommended to use let and const for superior memory management and bug avoidance.
