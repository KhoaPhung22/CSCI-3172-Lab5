## Logic and Reasoning
### 1. Data Structure (Maps and Sets)
- **Map (`userMap`)**: Used for stored credentials. Maps provide high-performance key-value lookups, allowing the `verifyUser` function to check credentials in `O(1)` time regardless of the number of users.
- **Set (`usernameSet`)**: Used to track existing usernames. This allows for near-instant checks for "username already exists" during registration, improving the efficiency over traditional array searching.

### 2. Validation Logic
- **Email**: Verified using regular expressions to ensure a traditional format and a Top-Level Domain (TLD) length between 2 and 8 characters.
- **Username**: Must not begin with a number and cannot contain spaces or special characters.
- **Password**: Must be at least **12 characters long** and include at least one uppercase letter, one lowercase letter, one number, and one special character.
- **Confirm Password**: Must exactly match the password field.

### 3. ES6 Features
- **Arrow Functions**: Used throughout the project (e.g., in `db.js`, `validation.js`, and event handlers) for concise and modern syntax.
- **Destructuring Assignment**: 
  - Used to extract form values from data objects efficiently.
  - Used in the `validateRegistrationForm` helper to handle multiple input fields simultaneously.
- **Modules**: The code is separated into `db.js` (data), `validation.js` (logic), and page-specific scripts to maintain a clean and scalable architecture.

### 4. Error Handling
- **Try-Catch Blocks**: Implemented in both the registration and login submission handlers. This makes the application safer by catching unexpected runtime errors and providing user-friendly feedback rather than crashing or failing silently.
### 5. Visual Feedback
- **Real-time Feedback**: Uses `blur` and `input` Event Listeners to validate fields as the user interacts with the form.
- **Visual Cues**: Border colors change dynamically (red for invalid, green for valid) to guide the user.
- **Notifications**: A custom notification system provides success/failure messaging at the top of the screen.
